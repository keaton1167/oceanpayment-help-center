- 待审核数。
- 待发布数。
- 已上线数。
- 资源异常数。
- 英文待复核数。

- 最近更新文档。

- 最近构建任务。

页面布局：

- 左侧目录树。
- 右侧目录详情表单。
- 底部影响范围提示。

目录字段：

- 中文名称。
- 英文名称。
- slug。
- 层级。
- 父目录。
- 排序。
- Docusaurus path。
- 启用状态。

主要操作：

- 新增一级目录。
- 新增二级目录。
- 编辑目录。
- 调整排序。

- 删除空目录。
- 预览 sidebar 结构。

## 5. 飞书导入

目标：让运营人员从飞书文档或知识库创建帮助中心文章。

### 5.1 单篇文档导入

操作流：

```text
粘贴飞书文档链接
  -> 系统解析标题和 token
  -> 选择目标语言和目录
  -> 生成 slug 和摘要建议
  -> 预览转换后的 MDX
  -> 下载图片和附件
  -> 创建草稿
  -> 进入内容管理编辑
```

页面字段：

- 飞书文档链接。
- 目标语言。
- 一级目录。
- 二级目录。
- slug。
- 是否生成英文草稿。
- 导入预览。
- 资源下载结果。

### 5.2 知识库批量导入

第一版如果开发资源紧张，可作为 P1。

操作流：

```text
粘贴知识库链接
  -> 读取知识库目录树
  -> 选择需要导入的节点
  -> 批量设置目标目录映射
  -> 执行导入任务
  -> 查看成功/失败列表
```

## 7. Agent 助手

目标：体现 Agent 应用落地，但第一版不必做成纯聊天入口。

建议第一版嵌入以下场景：

- 导入飞书文档后，Agent 生成摘要和标题建议。
- 编辑文档时，Agent 检查格式和链接。
- 中文文档生成英文草稿。
- 构建失败时，Agent 转换为运营可读说明。
- build 包生成后，Agent 生成飞书交接文案。

可独立页面展示：

- 快捷任务按钮。
- 最近 Agent 任务。
- 异常解释。
- 待处理建议。

快捷任务：

- 检查选中文档。
- 生成英文草稿。
- 生成部署交接说明。
- 汇总本周文档变更。
- 查找资源异常。

## 9. 系统配置

目标：维护非敏感配置，敏感配置只显示状态。

可编辑配置：

- 帮助中心正式域名。
- Docusaurus 项目路径。
- 中文 docs 目录。
- 英文 i18n 目录。
- 图片目录。
- 附件目录。
- build 输出目录。
- 默认通知群名称。

只显示状态：

- Feishu App ID 是否已配置。
- Feishu App Secret 是否已配置。
- 飞书登录回调是否可用。
- 机器人消息权限是否可用。
- OpenAI/模型服务是否已配置。

禁止：

- 页面明文展示 App Secret。
- 页面明文展示模型 API Key。
- 页面明文展示 SSH Key 或 GitHub Token。

## 11. 关键操作流

### 11.1 新建文章

```text
内容管理
  -> 新建文档
  -> 填写基础信息
  -> 编辑正文和附件
  -> 保存草稿
  -> 提交审核
  -> 审核通过
  -> 进入待发布
```

### 11.2 飞书导入文章

```text
飞书导入
  -> 粘贴文档链接
  -> 选择目录和语言
  -> 系统转换 MDX
  -> 运营预览和修正
  -> 保存草稿
  -> 提交审核
```

### 11.3 生成英文版本

```text
中文文档详情
  -> 点击生成英文草稿
  -> Agent 生成英文 MDX
  -> 标记英文待复核
  -> 人工检查
  -> 提交审核
```

### 11.4 build 包交付

```text
发布中心
  -> 查看待发布文档
  -> 构建检查
  -> 修复异常
  -> 生成 build.zip
  -> 生成变更清单
  -> 预览飞书通知
  -> 发送给技术支持部
  -> 标记已交付
  -> 技术支持部部署后标记已上线
```

## 12. 视觉方向建议

该系统是内部运营工具，视觉上应比营销官网克制，但不能过于粗糙。

建议：

- 使用清晰的信息密度，优先表格、状态、日志、任务反馈。
- 主色可延续 Oceanpayment 蓝绿体系，不照搬飞书默认蓝。
- 页面重点是“现在还有什么没处理”，而不是装饰。
- 异常和阻塞状态要明显。
- Agent 能力用“建议卡片”和“任务结果”呈现，避免做成无边界聊天框。
