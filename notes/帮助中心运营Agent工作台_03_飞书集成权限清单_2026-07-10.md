# 帮助中心运营 Agent 工作台飞书集成权限清单

更新日期：2026-07-10  
参考截图：飞书开放平台企业自建应用“帮助中心管理系统”机器人配置  
目标：复用现有企业自建应用，去除妙搭 APaaS 强依赖

## 1. 集成目标

新系统需要继续兼容飞书生态，但不再依赖妙搭作为运行平台。飞书在新系统中的角色是：

- 登录入口。
- 用户身份来源。
- 飞书文档和知识库内容来源。
- 图片和附件下载来源。
- 技术支持部 build 包交接通知渠道。
- 后续 Agent 机器人交互入口。

## 2. 需要复用的现有能力

从当前截图和旧系统结构看，建议继续复用：

- 企业自建应用：帮助中心管理系统。
- 机器人：帮助中心系统机器人。
- 网页应用能力。
- 权限管理中的云文档、知识库、消息、通讯录相关权限。
- 事件与回调能力，后续用于机器人消息和同步触发。

## 3. 必需权限清单

### 3.1 登录与用户识别

用途：

- 飞书登录。
- 获取当前用户身份。
- 映射系统角色。

需要确认的权限：

| 能力 | 用途 | MVP 是否必须 |
| --- | --- | --- |
| 获取用户基本信息 | 展示姓名、头像、邮箱 | 是 |
| 获取用户 open_id / union_id | 绑定系统用户 | 是 |
| 获取用户邮箱 | 权限和通知识别 | 是 |
| 获取部门信息 | 后续按部门授权 | 建议 |

### 3.2 云文档读取

用途：

- 导入单篇飞书文档。
- 读取标题、正文块、表格、链接。
- 生成 MDX 草稿。

需要确认的权限：

| 能力 | 用途 | MVP 是否必须 |
| --- | --- | --- |
| 读取云文档元信息 | 获取标题和文档类型 | 是 |
| 读取云文档内容块 | 转换正文 | 是 |
| 读取文档图片 token | 下载图片 | 是 |
| 读取文档附件 token | 下载附件 | 是 |

### 3.3 知识库读取

用途：

- 后续批量导入知识库。
- 读取知识库空间、节点树、节点对应文档。

需要确认的权限：

| 能力 | 用途 | MVP 是否必须 |
| --- | --- | --- |
| 读取知识库空间 | 选择知识库 | P1 |
| 读取知识库节点树 | 批量导入目录 | P1 |
| 读取节点对应文档 | 批量导入内容 | P1 |

说明：用户要求完整闭环中包含知识库批量导入，但如果开发资源有限，可先把单篇文档导入作为 P0，把知识库批量导入作为 P1。若必须第一版就完整覆盖，则该部分也升为 P0。

### 3.4 云盘/资源下载

用途：

- 下载文档图片。
- 下载附件。
- 可选：上传 build 包到飞书云盘后发送链接。

需要确认的权限：

| 能力 | 用途 | MVP 是否必须 |
| --- | --- | --- |
| 下载图片资源 | 文档图片落地到 `static/img/help-center` | 是 |
| 下载附件资源 | 附件落地到 `static/files/help-center` | 是 |
| 上传文件到云盘 | build 包过大时发送云盘链接 | 建议 |
| 创建分享链接 | 技术支持部下载 build 包 | 建议 |

### 3.5 机器人消息

用途：

- 向技术支持部群发送 build 包交接消息。
- 向审核人发送待审核提醒。
- 后续支持机器人指令。

需要确认的权限：

| 能力 | 用途 | MVP 是否必须 |
| --- | --- | --- |
| 发送群消息 | build 包交接 | 是 |
| 发送单聊消息 | 审核提醒 | 建议 |
| 上传消息附件 | 发送小文件 build 包或清单 | 建议 |
| 接收群消息事件 | 后续机器人指令 | P2 |
| 接收单聊消息事件 | 后续机器人指令 | P2 |

### 3.6 事件与回调

用途：

- 飞书登录回调。
- 后续机器人消息事件。
- 后续审批或任务提醒事件。

需要配置：

| 配置项 | 用途 | MVP 是否必须 |
| --- | --- | --- |
| 网页应用回调 URL | 飞书登录 | 是 |
| 机器人事件回调 URL | 接收机器人消息 | P2 |
| 加签密钥/Encrypt Key | 事件安全校验 | 按飞书配置要求 |

## 4. 需要向飞书开放平台确认的信息

进入开发前建议整理：

- App ID。
- App Secret 是否可由后端环境变量配置。
- 当前应用是否已启用机器人。
- 当前应用是否已启用网页应用。
- 当前已有权限列表截图或导出。
- 技术支持部通知群 chat_id。
- 审核提醒群或审核人 open_id。
- 登录回调域名，取决于管理系统部署地址。
- 事件回调域名，取决于管理系统部署地址。

## 5. 新系统配置建议

敏感配置放服务器环境变量：

```text
FEISHU_APP_ID
FEISHU_APP_SECRET
FEISHU_VERIFICATION_TOKEN
FEISHU_ENCRYPT_KEY
FEISHU_REDIRECT_URI
FEISHU_BOT_NOTIFY_CHAT_ID
```

非敏感配置可放系统配置页：

```text
帮助中心正式域名
Docusaurus 项目根目录
中文 docs 目录
英文 i18n 目录
图片目录
附件目录
build 输出目录
默认通知群名称
```

## 6. 旧 Miaoda 依赖替换表

| 旧能力 | 旧实现 | 新实现建议 |
| --- | --- | --- |
| 登录 | `@lark-apaas/fullstack` | 飞书 OAuth / 网页应用登录 |
| 用户信息 | APaaS 用户上下文 | Feishu OpenAPI + 本地 users 表 |
| 数据库 | APaaS 数据库封装 | PostgreSQL + ORM |
| 文件服务 | APaaS 文件服务 | 本地/对象存储/飞书云盘 |
| 文档读取 | 旧 Feishu service | 新 FeishuDocService |
| 知识库读取 | 旧 Feishu mappings | 新 FeishuWikiService |
| 机器人消息 | 旧平台能力 | FeishuBotService |

## 7. 权限风险

- 如果机器人没有文档访问权限，导入会失败。
- 如果文档图片/附件下载权限缺失，MDX 能生成但资源会缺失。
- 如果系统部署域名未确定，飞书登录回调无法最终配置。
- 如果 build 包通过飞书直接发送，可能遇到文件大小限制。
- 如果 App Secret 由多人手动传递，存在安全风险，应由部署人员写入服务器环境变量。
