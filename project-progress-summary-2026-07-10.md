## 3. 新阶段：帮助中心运营 Agent 工作台

### 3.1 产品定位

新系统定位为：

```text
帮助中心运营 Agent 工作台
```

它不是单纯后台，也不是纯聊天机器人，而是：

- 给运营人员使用的内部管理系统。
- 通过飞书登录识别用户。
- 支持飞书文档/知识库导入。
- 支持将飞书内容转换为当前帮助中心样式的 MDX。
- 支持审核、构建、build 包生成和飞书通知。
- 嵌入 Agent 能力做内容生成、质检、翻译草稿、构建错误解释和交接文案生成。

### 3.2 当前已明确边界

- 新系统建议新增独立目录：`help-center-admin/`。
- 不把管理后台混入 Docusaurus 公开站点。
- 不把管理系统打进公开帮助中心部署包。
- 旧 `miaoda-app/` 只作为参考，不迁移数据，不继续作为正式系统开发目录。
- 飞书企业自建应用和机器人能力可复用。
- 新系统应直接调用飞书 OpenAPI，不再依赖妙搭 APaaS。
- 系统第一版不自动部署公司服务器，仍由技术支持部手动部署 build 包。
- AI/Agent 第一版不能跳过人工审核。

### 3.3 建议目录结构

后续开始开发时，建议新增：

```text
help-center-admin/
  client/
  server/
  packages/
    help-center-core/
```

建议用途：

- `client/`：管理后台前端。
- `server/`：管理后台后端。
- `packages/help-center-core/`：Docusaurus 扫描、MDX 转换、资源检查、build 打包等通用能力。
