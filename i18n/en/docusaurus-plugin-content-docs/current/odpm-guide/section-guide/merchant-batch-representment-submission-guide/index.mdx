## Introduction to Batch Representment Functionality

### 1. Feature Overview

The bulk representment feature addresses merchants' customized requirements for representment materials, enabling them to independently define the layout and content of representment documents. Simultaneously, batch operations enhance the merchant experience by streamlining the process and reducing the complexity of multi-step procedures.

### 2. The Difference Between Batch Representments and Initiating Representments

Initiate a Re-presentment: To submit a representment for a chargeback order, click "Re-representment" under the chargeback order to access the designated representment submission page. Edit the text in the system-provided fields and upload supporting documentation.

Batch Representment: This is the functional button for the chargeback transaction section, enabling merchants to submit representment materials for multiple payment IDs. It also reduces system restrictions, allowing merchants to customize and edit representment documents.

Batch representments and individual representments are both methods for submitting representment materials, but the final representment outcome remains subject to the bank's decision.

### 3. Operation Path

ODPM > Payment > All Exceptions > Chargeback. Access the Chargeback Transactions section. On the current page, you will see the function button "Batch Representment".

### 4. Batch Representments Guidelines

To ensure representment documents are accurately matched to the chargeback orders requiring representment, follow the representment instructions. If the following conditions are met, you may submit a batch representment request.

- Before uploading files, compress them into ZIP format.
- The compressed file naming format must be: `Representment-YYYYMMDD.zip`.
- Within the compressed file, representment documents must be named using the payment ID.
- Duplicate payment ID representment submissions are not supported. If two payments with identical payment IDs require representment, only a single submission is permitted.
- For document formatting requirements, refer to the sample document.
- After submission, if the corresponding payment ID's chargeback status changes to "Re-presentment", it indicates that the document upload was successful.
- After successful upload, click "Download Voucher" to preview the representment document.

#### 5.1 Representment Format

- Text description + supporting images.
- Text must be in English.
- Font: 10 pt Microsoft YaHei.

#### 5.2 Representment Image

- Representment documentation images must be clear and appropriately sized.
- Formats: JPG, JPEG, or PNG.
- Maximum size per image: 3 MB.
- Maximum dimensions: 1800 px x 1500 px.

#### 5.3 Representment Document Restrictions

- The total number of pages in the Word document shall not exceed 20 pages.
- File size must not exceed 15 MB.
- The final Word representment document submitted must be named using the payment ID.
- The text descriptions in the required fields of the template are provided for reference. Optional sections may be supplemented or adjusted as needed based on actual circumstances.
- Before submitting materials, delete the Chinese text and supplementary highlighted content from the sample template.

### 6. How to Submit Batch Representments

- Click "Batch Representment".
- Click the upload icon on the pop-up page, then select the prepared compressed file.
- Click "Submit".
- After successful submission, click "Download Voucher" to review the submitted materials.

### 7. Upload Button Restriction Rules

- Compressed file naming: `Representment-YYYYMMDD.zip`. Example: `Representment-20250820.zip`.
- Maximum number of compressed packages: 1.
- Maximum number of files in a compressed package: 1000.
- Individual file size within the compressed package: 10 MB.
- Within the compressed file, representment documents for duplicate payment IDs are not permitted to be uploaded.
- For multiple chargebacks under the same payment ID, batch representments are not supported. Representments can only be initiated on a per-transaction basis.
- The payment ID must have a Chargeback status of "Unresolved".

### 8. Frequently Asked Questions (FAQ)

**Q: How many transactions can I dispute at one time?**

**A:** A single ZIP file upload supports up to 1,000 files, i.e. 1,000 payment IDs.

**Q: After uploading a file, how can I confirm that the submission was successful?**

**A:** Locate the payment ID for which you just submitted a representment. Check its "Chargeback Status". If the status has changed from "Unresolved" to "Re-presentment", this indicates that your document upload was successful and has been received by the system.

**Q: What should I do if I mistakenly submitted the wrong file or need to provide additional materials?**

**A:** This system does not support resubmitting the same payment ID through batch representment. If an error occurs during the operation or you need to add information, you may resubmit by clicking the "Re-presentment" button under the Operation column for the corresponding payment ID. Upon successful resubmission, the previous representment content will be overwritten, with the latest update time taking precedence.

**Q: What do you see when you click "Download Voucher"?**

**A:** This is a confirmation receipt generated by the system for your submission, presented in PDF format for your records and verification.

**Q: Can I upload documents in other formats such as PDF, JPG, or TIFF?**

**A:** Not supported. Currently only `.docx` files are supported.
