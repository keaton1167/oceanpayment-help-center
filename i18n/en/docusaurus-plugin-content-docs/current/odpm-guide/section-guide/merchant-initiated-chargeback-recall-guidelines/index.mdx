## 1. Difference Between Chargeback Recall and Representment

- Chargeback Representment is initiated by merchants who dispute a chargeback by  providing  evidence  that  the  chargeback  is  invalid  or  unjustified,  with  the outcome determined by the issuing bank.

- Chargeback Recall is a process where consumers proactively cancel a chargeback with the issuing bank and agree to return the funds. Merchants must supply the necessary recall documentation for the acquiring bank to confirm the status with the issuing bank.

## 2. Operation Path

Navigate to ODPM\> Payment \> All Exceptions \> Chargeback. The [Chargeback Recall Status] filter is available within the filter options.

![image17.png](/img/help-center/merchant-initiated-chargeback-recall-guidelines/image17.png)

## 3. Conditions for Initiating Chargeback Recall

- Depending on the chargeback process for different card types, the chargeback recall function may not be fully applicable to all orders. A chargeback recall

- request can be initiated if all the following conditions are met. Otherwise, the chargeback recall button will not be displayed.

- Payment method is Credit Card, GooglePay or ApplePay.

- Chargeback amount is greater than 0.

- Chargeback type is classified as 'chargeback'.

- Chargeback status is either 'Unresolved' or 'No response required'.

- The request is made within 15 days of the chargeback date.

- The chargeback recall status is 'Not applied', or it has been more than 7 days since the first unsuccessful recall attempt.

## 4. How to Initiate a Chargeback Recall Application

- Set the recall status filte rto 'Not Applied'.

- Click on the Operation and select "Chargeback Recall" to access the chargeback recall proof submission page.

![image18.jpeg](/img/help-center/merchant-initiated-chargeback-recall-guidelines/image18.jpeg)

- Proof of chargeback recall (required) : Supports formats doc, docx, pdf, jpg,

- Communication record (optional) : Same format support as for proof.

- File limit : 3MB per file, with a maximum of 5 files allowed.

- Initiation Timeframe: Within 15 days from the date of the chargeback.

- Number of submissions: Up to 2 attempts are permitted, with at least 7d ays between attempts.

- Supported Order Initiation Quantity: Only single transactions can be initiated; batch initiation is not supported.

![image19.jpeg](/img/help-center/merchant-initiated-chargeback-recall-guidelines/image19.jpeg)

## 5. Feedback on Chargeback Recall Outcome:

- Feedback provided within 5 business days.

- Recall Failed: If the consumer does not cancel thechargeback, the current status ismaintained (a second attempt can be initiated after 7days).

- Recall successful: If the consumer cancels the chargeback, the funds are returned through the successful represent mentstatus.
